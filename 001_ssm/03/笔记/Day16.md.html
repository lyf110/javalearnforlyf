```java
@Service
public class StockPositionServiceImpl implements IStockPositionService {

   @Autowired
   private ITradePositionDao tradePositionDao;

   @Autowired
   private RedisCacheManager redisCacheManager;

    /**
    * 获取所有持仓数据
    * @return
    */
   @Override
   public List<TradePosition> getAllPositions(long accountId) {
       // 根据账户ID获取所有持仓数据
       List<TradePosition> positionList = tradePositionDao.getAllByAccountId(accountId);
       if(null != positionList) {
           positionList.forEach(pos -> {
               // 计算浮动盈亏
               calcProfit(pos);
           });
       }
       return positionList;
   }


   /**
    * 获取股票产品的最新报价
    * @param stockId
    * @return
    * @throws ComponentException
    */
   private TradeStockQuoteLast getTradeStockQuoteLast(Long stockId){

       // 从缓存获取数据
       Cache lastPriceCache = redisCacheManager.getCache(GlobalConstants.STOCK_QUOTE_LAST_PRICE_KEY);
       Cache.ValueWrapper quoteLastWrapper = lastPriceCache.get(stockId);
       TradeStockQuoteLast quoteLast = null;
       if(quoteLastWrapper != null ) {
           quoteLast = (TradeStockQuoteLast) quoteLastWrapper.get();
       }

       return quoteLast;
   }



   /**
    * 计算仓位的浮动盈亏
    * @param position
    */
   private void calcProfit(TradePosition position) {
       // 获取最新行情报价
       TradeStockQuoteLast  quoteLast = getTradeStockQuoteLast(position.getStockId());
       if(null != quoteLast) {
           // 为防止客户端出现科学计数类型，这里采用长整型计算
           // 浮动盈亏（保留两位小数） = （最新价 - 成本价） * 持仓数量
           long diffPrice = MathUtils.sub(quoteLast.getLastPrice(), MathUtils.round2Long(position.getCostPrice()));
           long profit = MathUtils.mul(diffPrice, position.getVolume());
           position.setProfit(profit);
           position.setLastPrice(MathUtils.div(quoteLast.getLastPrice(), 100, 2));
           // 浮动盈亏比例（保留四位小数） = 浮动盈亏 / （成本价 * 持仓数量）
           position.setProfitRate(MathUtils.div2Long(profit, MathUtils.mul(position.getCostPrice(), position.getVolume()), 4));
       }
   }

}

```

  1. 先根据账号ID查询用户所有持仓信息， 再遍历通过，calcProfit方法计算浮动盈亏。

  2. 计算浮动盈亏，需要获取最新的行情报价， 前面我们在行情服务里面做了缓存， 这里通过getTradeStockQuoteLast方法获取。

  3. 为防止客户端出现科学计数类型，这里采用长整型计算， 建议都采用长整型计算， 双精度计算会存在精度丢失的可能。这个例子就会存在科学计算的问题：

    ```java

    System.out.println(MathUtils.div(10040, MathUtils.mul(87.8402, 200000, 4), 4))
    ```

3.2 账户资金总览功能计算公式介绍

账户资金总览， 从账户的资金维度， 统计股票与资金的关联概要性数据， 主要包含， 用户的总资产、总市值、持仓盈亏、当日持仓盈亏， 可用资金等。

  1. 用户的总资产是指用户的可用资金与所有股票市值的总和， 计算公式： 总市值 + 可用资金。

  2. 总市值是指用户所有持仓股票的最新价值总和， 计算公式： ∑ 股票最新价*持仓股数。

  3. 持仓盈亏是根据当前行情价， 计算用户所有持仓的盈利与亏损情况，

计算公式：∑ （现价-持仓成本价）*持仓股数。

  1. 当日持仓盈亏， 是指当天股票价格变化， 对用户持仓产生的盈利与亏损情况，

    计算公式：原持仓当日收益 + ∑当日卖出收益 + ∑当日买入收益

具体计算：

原持仓数量= 当前仓位总数量+ 当日卖出数量-当日买入数量）* (最新价格-昨日收盘价) + ∑
(成交价格 -作收价格) * 卖出数量 + ∑ (最新价格-成交价格) * 买入数量。

下面， 我们实现资金总览功能接口， 计算提供以上数据。

               // 更新收盘价(收盘5分钟内进行更新)

* 更新收盘价

private void updateClosePrice(TradeStockQuoteLast quoteList) {
// 判断是否到达收盘时间
if(isCloseTime(quoteList.getMarketNo())) {
// 当天收盘价
Cache curLastPriceCache = redisCacheManager.getCache(GlobalConstants.STOCK_QUOTE_CLOSE_PRICE_CUR_KEY);
// 昨日收盘价
Cache lastPriceCache = redisCacheManager.getCache(GlobalConstants.STOCK_QUOTE_CLOSE_PRICE_LAST_KEY);
// 更新作收价
if(null != lastPriceCache.get(quoteList.getStockId())) {
String lastClosePriceStr = (String)lastPriceCache.get(quoteList.getStockId()).get();
String[] lastClosePriceArr = lastClosePriceStr.split(“\|”);
if(!DateUtils.getFmtToday().equals(lastClosePriceArr[1])){
// 首次修改， 则更新, 避免重复更新
Long curLastPrice = (Long)curLastPriceCache.get(quoteList.getStockId()).get();
// 将当天收盘价更新为昨日收盘价, 附带时间标记， 下一天收盘需要再次更新
lastPriceCache.put(quoteList.getStockId(), curLastPrice + “|”+ DateUtils.getFmtToday());
log.info(“update lastPrice cache, stockId:{}, value:{}”, quoteList.getStockId(), curLastPrice + “|”+ DateUtils.getFmtToday() );
}

           }else {
               // 不存在， 则将当前收盘价放入缓存
               lastPriceCache.put(quoteList.getStockId(), quoteList.getLastPrice()+"|" + DateUtils.getFmtToday());
           }
           // 更新当前收盘价
           curLastPriceCache.put(quoteList.getStockId(), quoteList.getLastPrice());
       }
   }

  • 大家思考下，这里要保存作收价，但我们接收的是当前行情报价，如何在这里生成？

  • 我们可以每天都生成收盘价， 但是会造成缓存泛滥， 那设置超时时间？那超时时间设置多久？ 24小时处于边界， 不合适，改成48小时？ 这样是可以， 但是如果系统出现问题在几天后才发现， 想要再去修复处理， 就会比较麻烦， 这里我们的方案是只维护当天和上一天的收盘价， 做两个缓存。

  • 在更新收盘价的时候， 将作收价缓存改成当天收盘价缓存的值， 将当天收盘价缓存改成当前收盘价的值， 例如：

    10号 15:30 -> 更新收盘价为20.22 （cur：20.22, last：20.12）

    11号 15:30 -> 更新收盘价为20.32 将last 改成 cur的值， 将cur改成最新值。

  • 为避免多次调用造成的覆盖， 产生数据错误， 因为有5分钟的缓冲时间， 所以更新昨收价的时候， 还必须带上当天的日期作为标记， 如果当天的日期匹配， 则不再更新， 确保每天只更新一次。
