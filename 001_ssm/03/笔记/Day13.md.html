3.2 编写proto脚本

proto脚本分为两部分， 一部分是内部微服务gRpc通讯， 另一部分是外部客户端Netty长连接通讯。

以简易的下单接口作为示例， 了解proto使用以及通讯数据结构与规范。

  1. gRpc下单proto数据定义，StockTradeDealService.proto文件：

```protobuf
syntax = “proto3”;
option java_multiple_files = true;
option java_package = “com.itcast.bulls.stock.struct.grpc.trade”;
option java_outer_classname = “StockTradeDealServiceProto”;

// 交易服务的业务接口
service StockTradeDealService {

   //  定义下单接口
   rpc placeOrder (PlaceOrderRequest) returns (PlaceOrderReponse) {

   }

}

// 下单的请求参数
message PlaceOrderRequest {
// 股票ID
int64 stockId = 1;
// 股票的买卖方向(0: 买入， 1: 卖出）)
int32 direction = 2;
// 委托类型(0:市价单， 1:限价单)
int32 type = 3;
// 请求数量/挂单数量
int32 execvolume = 4;
// 请求价格/挂单/委托价格
double execprice = 5;
}

// 下单的返回对象
message PlaceOrderReponse{
// 状态，处理结果的状态
int32 status = 1;
// 处理结果的提示消息
string message = 2;
// 订单的ID
int64 dealId = 3;
// 成交的数量
int32 execvolume = 4;
// 成交的价格
double execprice = 5;
}

```

定义了placeOrder下单接口， 下单请求与下单返回数据的结构。

  1. Netty外部proto数据定义:
  • StockHead.proto文件：

    ```protobuf
option java_package = “com.itcast.trade.stock.struct.netty.trade”;

// 头部
message StockHead {

   ProtoVer protoVer = 1 ;
   RequestType requestType = 2;
   int64 seqId =3 ;
   int32 status =4;

// 协议版本

}

// 请求类型
enum RequestType {
// 用户登陆
USER_LOGIN = 0;

}
```

  • StockMessage.proto文件
    ```protobuf
    syntax = “proto3”;

option java_multiple_files = true;

import “StockHead.proto”;
import “StockQuote.proto”;
option java_package = “com.itcast.trade.stock.struct.netty.trade”;
option java_outer_classname = “StockMessageProto”;

// 请求消息
message StockMessage {

   // 头部消息
   StockHead stockHead = 1;

   // 消息体
   oneof StockData {
       PlaceOrderData placeOrderData = 2;
   }

}

// 返回消息
message ResponseMessage{
// 头部消息

// 下单请求数据包
```

3.4 封装编译脚本

最新Protobuf不支持通配符命令， 不能同时编译多个文件， 这里我们通过自定义脚本来支持， 制定好脚本也能便于对数据结构的维护。

  1. protoc_grpc.bat文件

```shell
@echo off
for %%i in (grpc/*.proto) do (
d:/TestCode/protoc.exe –plugin=protoc-gen-grpc-java=d:/TestCode/protoc-grpc.exe –java_out=../../java –grpc-java_out=../../java ./grpc/StockKlineService.proto
echo generate %%i to java file successfully!
)

```

  1. protoc_netty.bat文件

```shell
@echo off
for %%i in (netty/*.proto) do (
d:/TestCode/protoc.exe –proto_path=./netty –java_out=../../java ./netty/%%i
echo generate %%i to java file successfully!
)

```

com.itcast.trade.stock.deal.service.GrpcStockDealService类：

```java
@GrpcService
@Log4j2
public class GrpcStockDealService extends StockTradeDealServiceGrpc.StockTradeDealServiceImplBase {

   @Autowired
   private ITradeOrderDao tradeOrderDao;

   /**
    * <pre>
    * place order
    * </pre>
    */
   @Override
   public void placeOrder(PlaceOrderRequest request,
                          StreamObserver<PlaceOrderReponse> responseObserver) {
       log.info("Process in GrpcStockDealService.placeOrder, request: " + request);

       // 保存订单信息
       TradeOrder order = new TradeOrder();
       order.setDirection(request.getDirection());
       order.setStockId(request.getStockId());
       order.setAccountId(0l);
       order.setExecPrice(request.getExecprice());
       order.setExecVolume(request.getExecvolume());
       order.setStatus(0);
       order.setType(request.getType());
       order.setInitVolume(request.getExecvolume());
       tradeOrderDao.insert(order);

       // 设置返回信息
```

@Log4j2
public class StockProcessorFactory {

    /**
     * 交易服务的处理器
     */
    @Autowired
    private IBaseStockProcessor stockDealProcessor;

    /**
     * 用户服务的处理器
     */
    @Autowired
    private IBaseStockProcessor stockUserProcessor;

    /**
     * 行情服务的处理器
     */
    @Autowired
    private IBaseStockProcessor stockQuoteProcessor;

    /**
     * 根据请求参数获取指定的业务处理器
     * @param requestType
     * @return
     * @throws ComponentException
     */
    public IBaseStockProcessor generateStockProcessor(StockHeadProto.RequestType requestType) throws ComponentException {

        IBaseStockProcessor processor = null;
        switch(requestType) {
            case USER_LOGIN:
                processor = stockUserProcessor;
                break;
            case PLACE_ORDER:
                processor = stockDealProcessor;
                break;
            case QUOTE_KLINE:
                processor = stockQuoteProcessor;
                break;
            default:
                log.error("Process in generateStockProcessor, not found processor! ");
                throw new ComponentException(ApplicationErrorCodeEnum.NOT_FOUND_PROCESSOR);

        }
        return processor;
    }

}

```java
    * @param ch
    * @throws Exception

// 在调试期加入日志功能，从而可以打印出报文的请求和响应细节
if(debug) {
pipeline.addLast(new LoggingHandler());
}

       // 增加对HTTP SERVER的支持, 初始化对应的包大小
       pipeline.addLast(new HttpServerCodec());
       //HTTP聚合解析， 设置聚合内容的最大长度
       pipeline.addLast(new HttpObjectAggregator(Integer.MAX_VALUE));
       // 支持大数据流写入
       pipeline.addLast(new ChunkedWriteHandler());
       // 支持WebSocket数据压缩
       pipeline.addLast(new WebSocketServerCompressionHandler());
       // Websocket协议配置， 设置访问路径
       pipeline.addLast(new WebSocketServerProtocolHandler("/stock", null, true, Integer.MAX_VALUE));

       //Protobuf的解码器，通过Google Protocol Buffers序列化框架动态的切割接收到的ByteBuf
       pipeline.addLast(new ProtobufVarint32FrameDecoder());
       pipeline.addLast(new ProtobufVarint32LengthFieldPrepender());


       // 协议包解码
       pipeline.addLast(new MessageToMessageDecoder<WebSocketFrame>() {
           @Override
           protected void decode(ChannelHandlerContext ctx, WebSocketFrame frame, List<Object> objs) throws Exception {
               log.info("decode client data ------------------------");
               if (frame instanceof TextWebSocketFrame) {
                   TextWebSocketFrame textFrame = (TextWebSocketFrame)frame;
                   log.info("MsgType is TextWebSocketFrame");
               } else if (frame instanceof BinaryWebSocketFrame) {
                   ByteBuf buf = ((BinaryWebSocketFrame) frame).content();
                   objs.add(buf);
                   buf.retain();
                   log.info("MsgType is BinaryWebSocketFrame");
               } else if (frame instanceof PongWebSocketFrame) {
                   log.info("MsgType is PongWebSocketFrame ");
               } else if (frame instanceof CloseWebSocketFrame) {
                   log.info("MsgType is CloseWebSocketFrame");
                   ch.close();
               }

           }
       });
       // 协议包编码
       pipeline.addLast(new MessageToMessageEncoder<MessageLiteOrBuilder>() {
           @Override
           protected void encode(ChannelHandlerContext ctx, MessageLiteOrBuilder msg, List<Object> out) throws Exception {
               ByteBuf result = null;
               if (msg instanceof MessageLite) {
                   result = wrappedBuffer(((MessageLite) msg).toByteArray());
               }
               if (msg instanceof MessageLite.Builder) {
                   result = wrappedBuffer(((MessageLite.Builder) msg).build().toByteArray());
               }

               // 下面再转成websocket二进制流，因为客户端不能直接解析protobuf编码生成的数据
               WebSocketFrame frame = new BinaryWebSocketFrame(result);
               out.add(frame);
           }
       });
       // Protobuf的解码器， 转换为自定义的StockMessage
       pipeline.addLast(new ProtobufDecoder(StockMessage.getDefaultInstance()));


       // 自定义数据处理器
       pipeline.addLast(stockProxyServerHandler);

}

}
```

StockProxyServerHandler类：

```java
@Component
@Sharable
@Log4j2
public class StockProxyServerHandler extends SimpleChannelInboundHandler { 

   /**
    * Spring 的线程池
    */
   @Autowired
   private TaskExecutor taskExecutor;

   /**
    * 处理器工厂
    */
   @Autowired
   private StockProcessorFactory stockProcessorFactory;

   /**
    * 负责客户端的Channel管理(线程安全)
    */
   public static ChannelGroup channels = new DefaultChannelGroup(GlobalEventExecutor.INSTANCE);


   /**
    * 接收处理客户端发送数据
    * @param channelHandlerContext
    * @param stockMessage
    * @throws Exception
    */
   @Override
   protected void channelRead0(ChannelHandlerContext channelHandlerContext, StockMessage stockMessage) throws Exception {

       // 异步处理
       taskExecutor.execute(() ->{
           try {
               // 获取指定处理器，处理业务请求数据
               stockProcessorFactory.generateStockProcessor(stockMessage.getStockHead().getRequestType()).processRequest(channelHandlerContext, stockMessage);
           }catch(ComponentException e) {
               // 业务异常
               log.error(e.getMessage(), e);
               StockHeadProto.StockHead stockHead = generateStockHead(stockMessage.getStockHead(), GlobalConstants.STATUS_ERROR);
               ResponseMessage responseMessage = ResponseMessage.newBuilder().setStockHead(stockHead)
                       .setMessage(e.getErrorCodeEnum().getMessage()).build();
               channelHandlerContext.writeAndFlush(responseMessage);
           }catch(Exception e) {
               // 系统异常
               log.error(e.getMessage(), e);
               StockHeadProto.StockHead stockHead = generateStockHead(stockMessage.getStockHead(), GlobalConstants.STATUS_ERROR);
               ResponseMessage responseMessage = ResponseMessage.newBuilder().setStockHead(stockHead)
                       .setMessage(e.getMessage()).build();
               channelHandlerContext.writeAndFlush(responseMessage);
           }
       });

   }

   /**
    * 生成返回的头部信息
    * @param requestType
    * @param seqId
    * @param status
    * @return
    */
   protected StockHeadProto.StockHead generateStockHead(StockHeadProto.StockHead stockHead, String status) {

       StockHeadProto.StockHead stockHeadResult = StockHeadProto.StockHead.newBuilder()
               .setRequestType(stockHead.getRequestType()).setSeqId(stockHead.getSeqId())
               .setStatus(status).build();

       return stockHeadResult;
   }


   @Override
   public void handlerAdded(ChannelHandlerContext ctx) throws Exception {


   }

   @Override
   public void handlerRemoved(ChannelHandlerContext ctx) throws Exception {

       // ChannelGroup 会自动调用处理， 不写也可以
       channels.remove(ctx.channel());

       // 主动关闭
       ctx.close();
   }

   @Override
   public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
       log.error("exceptionCaught on socket", cause);
       ctx.close();
   }

}
```

  • channelRead0方法， 接收服务端发送的数据， 通过Protobuf解码器， 会自动转换成StockMessage对象。
  • 为提升处理效率， 增加异步线程池TaskExecutor处理。
  • 这里先不作鉴权， 下面再讲， 直接通过工厂方法，获取指定的业务处理器， 接收处理数据。
  • 针对业务异常与系统， 作不同的消息提示封装。
  • 这里采用了ChannelGroup， 负责客户端的Channel连接通道管理， 是线程安全；连接中断的时候自动释放， 作为长连接， 消息的推送和广播， 需要使用到。
